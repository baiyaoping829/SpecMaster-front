# 规标大师（specmaster）事故案例模块：事故管理子模块端到端测试方案（人工交互 + 可回放脚本）

## 目标与结论口径

- 目标：对“事故案例 > 事故管理”进行端到端功能验证，覆盖 UI 操作、前后端接口链路与关键边界输入。
- 结论口径：
  - 功能全部正常：关键路径均通过，且不存在阻塞级缺陷。
  - 存在阻塞性缺陷：关键路径不可用，或需求范围中关键能力未实现/不可验证。

## 人工交互用例（可回放脚本同名覆盖）

### 1) 基础可用性
- 打开“事故案例”模块，默认展示“事故管理”Tab
- Tab 切换：事故管理 ↔ 事故分析 ↔ 媒体报道
- 列表出现数据行（可由种子数据脚本生成）

### 2) 事故录入（必填校验）
- 点击“添加事故案例”
- 不填必填项直接保存：应提示
  - 事故名称/事故类型/事故日期/事故等级/所属省份为必填
- 逐项补齐后保存：应新增一条记录并可在列表搜索到

### 3) 编辑与删除（回归 CRUD）
- 编辑一条记录：修改事故名称并保存
- 列表应更新；搜索应命中
- 删除该记录：列表应移除

### 4) 附件上传（现状为 UI 占位）
- 在弹窗里为“事故调查报告/阅览报告/媒体报道”选择本地文件
- 期望：file-list 出现文件（仅 UI 验证，不验证后端存储）

### 5) 边界输入（200+）
- 通过脚本生成 200+ 事故条目，覆盖：
  - 超长标题（接近 256）
  - 特殊字符（空格、引号、emoji、CJK）
  - 空白/仅空格
  - 非法日期（API 层应拒绝；UI 层不一定覆盖）
- 在 UI 侧验证：分页/搜索性能与稳定性

## 自动化回放（Playwright）

```bash
npx playwright test -c playwright.accident.config.ts
```
