# 问题清单

## 1. 启动与运行

- P0｜前端“空白/黑屏”｜根因：开发服务器被错误传入位置参数，Vite 以错误 root 启动导致 `GET /` 返回 404，从而页面不渲染（运行时表现为空白页）。
- P1｜缺少可用后端与API联调通路｜现状：仓库以单体前端为主，`/api` 仅有 axios 封装与接口定义，但开发环境未配置代理且无后端实现，容易导致联调阶段大量 404/跨域。

## 2. 构建与类型系统

- P0｜`npm run build` 失败｜根因：`vue-tsc -b` 在严格模式下对大量页面文件报错（未使用变量/参数、可能为 `undefined` 的访问、隐式 any、数组越界风险等），导致无法产出生产构建。
- P1｜缺少 `*.vue` 模块声明｜表现：跨组件引入 `*.vue` 时 `TS7016`（无声明文件）导致类型检查失败。
- P1｜d3 类型缺失｜表现：`TS7016`（无声明文件）影响构建与IDE诊断。

## 3. 安全与供应链

- P0｜依赖漏洞：immutable < 5.1.5｜影响：存在已披露安全风险，需升级依赖树修复。
- P1｜PDF.js worker 外链与版本错配｜表现：使用 cdnjs 的 `pdf.js@4.0.379` worker 与项目 `pdfjs-dist@5.x` 主库不一致，存在渲染异常与外部依赖不可用/合规风险。

## 4. 架构与可维护性

- P1｜HTTP 响应契约假设过强｜表现：axios 拦截器硬编码 `res.code===200`，与常见 REST 直接返回 data 的形态不兼容，易导致“误判失败”。
- P1｜大型单文件组件体量过大｜表现：`src/views` 多个页面为超大 SFC，静态分析问题集中、边界条件更易遗漏，维护成本高。

## 5. 性能

- P2｜构建产物 chunk 过大警告｜表现：`vite build` 输出 chunk 体积超过默认阈值，可能影响首屏加载与缓存命中。
