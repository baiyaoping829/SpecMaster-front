### 2) 供应链安全修复

- 执行依赖审计并修复高危漏洞：immutable 依赖树升级至安全版本。
- 结果：`npm audit` 高危项清零。

### 3) PDF.js 稳定性修复

- 修复点：
  - worker 从外链改为随构建产物打包的本地 worker（避免外部依赖与版本错配）。
  - cMapUrl 改为基于运行时 `pdfjsLib.version` 动态选择 CDN 路径（避免主库与资源版本不一致）。
- 文件：[AccidentAnalysis.vue](file:///h:/learnproject/04%20solo_builder/src/views/accident-cases/AccidentAnalysis.vue)

### 5) 构建产物优化

### 6) HTTP 封装健壮性增强

- 兼容两类响应形态：
  - 后端统一包裹 `{ code, message, data }`
  - 直接返回 data（无 code 字段）
- 请求头防御：确保 `config.headers` 存在后再注入 `Authorization`。
- 文件：[request.ts](file:///h:/learnproject/04%20solo_builder/src/utils/request.ts)

## 使用说明（开发/生产）

- 开发（两进程）：
  - 后端：`npm run dev:api`（默认 `http://localhost:3001`）
  - 前端：`npm run dev`（`http://localhost:5173`，通过代理访问 `/api`）
- 生产（单进程，同源）：
  - 构建：`npm run build`
  - 启动：`npm start`（`http://localhost:3001`）
