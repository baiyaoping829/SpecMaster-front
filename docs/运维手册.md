# 运维手册

## 1. 部署顺序（建议）

1) PostgreSQL/MySQL（含主从/集群与故障转移组件）  
2) Redis（建议 Sentinel/Cluster，开启 ACL/TLS）  
3) MinIO（建议分布式 + 版本控制 + 站点复制）  
4) 应用服务（specmaster）  
5) Prometheus + Grafana + OTEL Collector（可选但建议）

## 2. 开发环境（Docker）

启动基础设施：

- `docker compose -f docker-compose.infra.yml up -d`

启动应用（生产同源模式）：

- `npm ci`
- `npm run build`
- `MINIO_ENABLED=1 REDIS_ENABLED=1 npm start`

## 3. 生产环境启动参数（示例）

应用启动（单端口同源）：

- `PORT=3001`
- `DB_DIALECT=postgres|mysql|sqlite`
- `AUTO_MIGRATE=0`（生产建议由 Flyway/Liquibase 管理迁移）
- `JWT_SECRET=...`（必须替换）

PostgreSQL：

- `PG_HOST=...`
- `PG_PORT=5432`
- `PG_DATABASE=specmaster`
- `PG_USER=specmaster_rw`
- `PG_PASSWORD=...`

MySQL：

- `MYSQL_HOST=...`
- `MYSQL_PORT=3306`
- `MYSQL_DATABASE=specmaster`
- `MYSQL_USER=specmaster_rw`
- `MYSQL_PASSWORD=...`

Redis：

- `REDIS_ENABLED=1`
- `REDIS_URL=redis://...`

MinIO：

- `MINIO_ENABLED=1`
- `MINIO_ENDPOINT=https://minio.internal`
- `MINIO_ACCESS_KEY=...`
- `MINIO_SECRET_KEY=...`
- `MINIO_BUCKET=specmaster-docs`
- `MINIO_PRESIGN_EXPIRES_SECONDS=300`
- `MINIO_PUBLIC_BASE_URL=`（如走 CDN，可配置为 `https://cdn.example.com`）

OpenTelemetry：

- `OTEL_EXPORTER_OTLP_ENDPOINT=http://otel-collector:4318/v1/traces`
- `OTEL_SERVICE_NAME=specmaster`

## 4. 数据库高可用建议

### PostgreSQL

- 方案：Patroni + etcd/consul（自动故障转移）或云厂商托管 HA
- 连接策略：写连接指向 VIP/主库代理；读连接指向只读副本池
- 监控：慢查询、连接池、复制延迟、死锁次数

### MySQL

- 方案：InnoDB Cluster（Group Replication）或 MGR + ProxySQL
- 连接策略：写连接指向主；读连接按副本池做负载均衡
- 监控：慢查询、InnoDB 行锁等待、复制延迟

## 5. Redis 安全与可用性

- 禁止公网暴露 6379，使用内网 + ACL + TLS
- 生产建议：Sentinel/Cluster，开启 AOF + RDB
- 告警：命中率 < 90%、内存水位、evicted_keys 增长、主从延迟

## 6. MinIO 高可用与合规

- 桶策略默认私有，通过预签名 URL 限时访问
- 开启版本控制（防覆盖/误删回滚）
- 多站点复制（异地容灾）
- 对敏感文件开启服务端加密（SSE-S3 或 KMS）

## 7. 迁移与回滚

### Flyway（推荐在 CI/CD 中执行）

- PostgreSQL 配置：[flyway-postgres-dev.conf](file:///h:/learnproject/04%20solo_builder/flyway/conf/flyway-postgres-dev.conf)
- MySQL 配置：[flyway-mysql-dev.conf](file:///h:/learnproject/04%20solo_builder/flyway/conf/flyway-mysql-dev.conf)

灰度策略：

- 新旧版本并行，先做只读流量验证，再逐步放量写流量
- 缓存策略：上线期间缩短 TTL，回滚时优先清缓存 key 前缀

## 8. 故障排查 SOP（摘录）

- API 5xx：先看 `/api/health` → DB/Redis/MinIO 哪个不可用
- 缓存不一致：检查 outbox 是否堆积（可临时关闭 consumer 再补偿），必要时运行一致性校验脚本
- 文件无法下载：检查 MinIO 存活、桶策略、预签名过期时间与系统时钟
