# 事故爬取子模块操作手册

## 1. 功能入口

前端：进入“事故案例”模块，在标签页中选择“事故爬取”。

## 2. 创建并运行爬取任务

1. 在“事故爬取”页顶部输入：
   - 关键词：如“重大事故 河北 2024”
   - 目标网址：建议为列表页或通报汇总页
2. 点击“创建任务”
3. 在任务列表中点击“运行”

任务状态说明：
- `pending`：已创建未执行
- `running`：执行中
- `succeeded`：执行完成且无失败
- `finished`：执行完成但存在失败（可查看 `last_error`）

## 3. 查看/检索已存事故摘要

在“已存事故摘要”区域支持：
- 关键词/地点检索
- 时间区间过滤
- 事故类型过滤
- 伤亡等级过滤
- 排序与分页

支持操作：
- “重解析”：对已存 HTML/TXT 报告重新提取字段
- “删除”：软删除该报告

## 4. 在新增事故案例中使用“案例选”

1. 打开“事故管理”→“添加事故案例”
2. 点击新增页面顶部“案例选”
3. 在弹窗中按条件检索并勾选目标案例
4. 选择：
   - “填充到表单”：将第一条勾选记录映射填充到表单
   - “批量创建”：将勾选记录批量写入事故案例库（自动生成案例编号）

字段映射（默认）：
- 事故名称 ← 报告标题
- 事故概要 ← 概述（overview）
- 事故类型 ← accident_type
- 直接原因 ← cause_overview
- 责任单位 ← responsible_party（为空则“外部爬取”）
- 事故日期 ← occurred_at
- 所属省份 ← location（截断）
- 人员伤亡 ← casualties
- 媒体报道链接 ← source_url/report_url

## 5. 存储位置

下载/缓存文件默认落在：

`backend/storage/crawler/`
- `raw/{task_id}/...`：HTML/PDF 原始文件
- `logs/*.log`：任务日志
